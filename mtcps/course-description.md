# Models and Tools for Cyber Physical Systems

## Description

The aim of this course is to introduce mathematical models for the formal description and analysis of programs, with emphasis on parallel and reactive systems. The course consists of several modules, each spanning 2-5 lectures, and two mini-projects. It deals with semantic models for parallel systems and logics for the description of their properties. As part of the course material, we also introduce several automatic verification tools and hint at some of the implementation techniques underlying them.

**Disclaimer:** This is an English translation of the module. In case of discrepancy between the translation and the Danish version, the Danish version of the module is valid.

**Purpose:** Cyber Physical Systems, systems that both contain embedded real-time software and consist of a physical component, are becoming more and more widely used. This course supports the semester project by presenting relevant methods and theories.

## Learning Objectives

### Knowledge

- Formal models for Cyber Physical systems
- Tools for simulation and analysis of functional, time and performance requirements
- Different languages for implementing real-time systems
- Real-time operating systems
- Embedded hardware
- Data collection and communication
- Dynamic and/or hybrid systems

### Skills

- Explain the special types of requirements that apply to Cyber Physical Systems
- Explain the definitions of the formal models of Cyber Physical Systems reviewed in the course

### Competences

- Reason about the choice of technologies for the development of a Cyber Physical System
- Demonstrate a coherent understanding of the various components that make up a Cyber Physical System
